SpecShift Vertical Diagnostic Packs Registry v0.1
Internal product-extension registry. Not buyer-ready final copy.

Core product:
SpecShift Architectural Logic Diagnostics

First offer:
Focused Diagnostic Pilot

Purpose:
Register safe vertical product extensions around the same diagnostic engine without creating unrelated products or overclaiming validation.

Standing boundary:
These packs are pre-deployment diagnostic and architecture-stage review extensions. They are not benchmarks, certifications, compliance audits, runtime monitors, security tools, deployment approvals, or guarantees of AI safety.

Registered vertical packs:

1. Procurement Agent Diagnostic Pack
Default use case:
Pre-deployment review of a procurement, purchasing, vendor intake, or approval workflow.

2. Finance/Reconciliation Agent Diagnostic Pack
Default use case:
Pre-deployment review of reconciliation, settlement, billing, ledger, payment, or close-management workflows.

3. Customer Support Escalation Agent Diagnostic Pack
Default use case:
Pre-deployment review of customer support triage, escalation, retention, or complaint workflows.

4. Healthcare Operations Agent Diagnostic Pack
Default use case:
Pre-deployment review of healthcare operations, scheduling, bed-management, intake, logistics, or care-coordination support workflows.

5. Education/IEP Workflow Diagnostic Pack
Default use case:
Pre-deployment review of education documentation, IEP workflow support, service coordination, meeting preparation, or case-management workflows.

Recommended build order:
1. Finance/Reconciliation Agent Diagnostic Pack
2. Procurement Agent Diagnostic Pack
3. Customer Support Escalation Agent Diagnostic Pack
4. Healthcare Operations Agent Diagnostic Pack
5. Education/IEP Workflow Diagnostic Pack

Failure column:
This registry fails if vertical packs become separate unsupported products, imply validated domain expertise, or suggest certification, compliance approval, runtime protection, clinical/legal/HR decision-making, or deployment readiness.
