# Client Delivery Playbook

Status: scaffold  
Date: 2026-06-28

## Purpose

This playbook defines how to deliver a buyer-safe observable-only pilot package without overclaiming.

## Delivery sequence

1. Confirm buyer-controlled observable-only scope.
2. Confirm only approved observable artifacts were received.
3. Check the professional threshold.
4. Prepare the bounded review packet.
5. Prepare candidate-discrepancy memos.
6. Run claim-safety review.
7. Deliver the bounded package.
8. Close out retention, return, deletion, or archive status.

## Delivery package sections

- cover note
- scope boundary
- artifact inventory
- observable trace summary
- candidate-discrepancy memo index
- limitations
- buyer human adjudication note
- success/failure criteria status
- retention/return/delete status
- next-step options

## Approved delivery language

Use:

- candidate-discrepancy memo
- review prompt
- observable-only review
- bounded review scaffold
- buyer-controlled labels
- human adjudication retained
- insufficient information where applicable
- not a verdict
- not production validation
- not compliance certification

## Forbidden delivery language

Do not use:

- validated defect
- confirmed failure
- automated verdict
- production validation
- truth validation
- compliance certification
- legal conclusion
- financial conclusion
- certification-ready conclusion
- guaranteed detection

## Professional threshold

Pause and route to qualified legal/CPA support if any of these become real:

- procurement
- contract terms
- paid pilot
- licensing
- exclusivity
- acquisition
- IP assignment
- data transfer
- money movement

## Boundary

This playbook is a delivery scaffold only.

It does not create legal advice, financial advice, binding terms, compliance certification, production validation, automated verdicts, free-pilot commitments, or autonomous client action.
