# SpecShift Buyer-Safe Pilot Intake + Delivery Lane Workplan

Status: registered  
Date: 2026-06-28

## Purpose

Build a buyer-safe pilot intake and delivery lane.

The core question is:

If a company says yes, what do they send us, where does it go, what do we return, and how do we avoid overclaiming?

## Expected outputs

- `client_intake_questionnaire.md`
- `buyer_data_request_template.md`
- `pilot_scope_and_boundaries.md`
- `candidate_discrepancy_memo_template.md`
- `pilot_delivery_checklist.md`
- `client_project_folder_generator.py`
- `client_delivery_playbook.md`
- `pilot_success_failure_criteria.md`
- `non_retention_and_boundary_statement.md`

## Required pilot boundaries

- buyer-controlled
- observable-only
- exported traces only
- no model weights
- no source code
- no private prompts
- no hidden activations
- no private chain-of-thought
- buyer retains labels
- human adjudication retained
- candidate-discrepancy memos only
- no automated verdict
- no production validation claim
- no legal/compliance/certification claim
- no free pilots

## Professional threshold

If procurement, contract terms, paid pilot, licensing, exclusivity, acquisition, IP assignment, data transfer, or money movement becomes real, engage qualified legal and CPA support before proceeding alone.

## Boundary

This workplan builds a buyer-safe pilot intake and delivery lane only.

It does not create legal advice, financial advice, binding terms, compliance certification, production validation, automated verdicts, or autonomous client action.
