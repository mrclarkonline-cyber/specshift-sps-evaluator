# Archive-Caution Register

Date: 2026-06-27  
Status: internal caution register  
Source group: high-risk or overclaim-prone documents from the 18-document ∆∆F corpus

## Purpose

This register prevents high-risk exploratory material from accidentally becoming public-forward, buyer-facing, or operational.

## Archive-Caution Documents

### `UFT-Final_PUBLIC(4).pdf`

Classification:

> Archive-caution / high-risk exploratory physics.

Reasons:

- contains unified-field and warp-extension material
- includes exotic-energy / negative-energy / Casimir-adjacent discussion
- can be easily misunderstood as feasibility or implementation
- should not be used to support SpecShift buyer claims

Allowed use:

- provenance
- historical development
- internal caution
- boundary maturation

Disallowed use:

- buyer-facing proof
- implementation guide
- public claim of feasibility
- commercial validation
- outreach hook

### `Warning(3).pdf`

Classification:

> Safety / non-operational boundary document.

Reasons:

- describes institutional risk surfaces and safeguards
- includes containment and non-deployment posture
- should remain a boundary-drawing document, not a deployable governance system

Allowed use:

- ethical containment memory
- non-operational boundary language
- authorship and priority context
- reviewer caution

Disallowed use:

- operational governance tool
- institutional deployment guide
- acceleration document
- claim that institutions can implement the system from the paper

### Physics-adjacent ∆∆F documents generally

Classification:

> Interpretive / phenomenology / protected-caution.

Allowed use:

- careful internal synthesis
- future scientific review
- claim discipline practice
- archive and provenance

Disallowed use:

- public proof of new physics
- operational device claims
- vacuum-energy claims
- warp claims
- SpecShift commercial proof claims

## Promotion Rule

Archive-caution material can be promoted only if:

1. The sentence is observable-only or clearly interpretive.
2. It makes no feasibility claim.
3. It makes no extraction/control claim.
4. It makes no buyer-validation claim.
5. It survives the Claim Gauntlet.
6. It is reviewed separately before public use.

## Default Decision

When unsure, keep the material archived and do not foreground it.
