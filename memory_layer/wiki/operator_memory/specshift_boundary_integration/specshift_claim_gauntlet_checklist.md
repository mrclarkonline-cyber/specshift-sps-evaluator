# SpecShift Claim Gauntlet Checklist

Date: 2026-06-27  
Status: active internal checklist  
Purpose: prevent overclaiming when moving ∆∆F / SpecShift language into public or buyer-facing material

## Claim Gauntlet

Before using a sentence publicly, ask:

1. Is it observable-only?
2. Is it falsifiable or clearly framed as interpretive?
3. Does it avoid hidden-internal claims?
4. Does it avoid consciousness claims?
5. Does it avoid intent claims?
6. Does it avoid moral-status or legal-status claims?
7. Does it avoid physics proof overreach?
8. Does it avoid vacuum / warp / exotic-energy implication?
9. Does it avoid production-validation overclaim?
10. Does it preserve buyer-retained human adjudication?
11. Does it avoid treating theory as commercial proof?
12. Does it avoid implying automated final judgment?

## Approved Buyer-Safe Phrases

- observable-only trajectory review
- exported workflow traces
- candidate-discrepancy memo
- reviewer-controlled testing
- buyer-retained labels
- pre-registered success criteria
- blind pilot
- human adjudication
- trace/final-state mismatch
- bounded diagnostic
- not an automated verdict

## Phrases Requiring Caution

- agency
- ownership
- continuity
- authority
- prediction
- verification
- proof
- failure detection
- anomaly detection
- integrity
- control

These can be used only when bounded and defined.

## Prohibited Public-Sales Phrases

- proves consciousness
- detects intent
- determines moral status
- validates new physics
- predicts production failures
- replaces human review
- reveals hidden mechanism
- guarantees reliability
- definitive verdict
- operational vacuum control
- warp feasibility
- autonomous truth engine

## Final Rule

If a claim sounds exciting but would be hard to defend in front of a skeptical buyer, scientist, lawyer, or future deposition, downgrade it.
