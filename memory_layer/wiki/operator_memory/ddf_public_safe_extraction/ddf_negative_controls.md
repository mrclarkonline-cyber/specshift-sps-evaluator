# ∆∆F Negative Controls

Date: 2026-06-27  
Status: internal extraction draft  
Source group: diagnostic-core documents from the 18-document ∆∆F corpus

## Purpose

Negative controls are required to keep ∆∆F / κ from becoming a claim-forward story.

They test whether apparent curvature structure disappears when the relevant organizing relationship is removed.

## Core Negative-Control Principle

A valid diagnostic should be quiet under conditions where the claimed structure is absent.

## Negative Control Types

### 1. Scrambled-control null

Preserve:

- marginal action/control distribution
- broad first-order statistics where possible

Destroy:

- temporal ordering
- state-action coupling
- causal sequence
- conditional structure

Expected behavior:

- observed and scrambled curvature should differ only if the original sequence contains structure tied to coupling or ordering.

### 2. Randomized directional null

Preserve:

- curvature magnitudes or related local statistics

Destroy:

- directional sign coherence
- persistent correction direction
- recurrence structure

Expected behavior:

- persistent directional organization should not survive random sign/direction reassignment.

### 3. Frozen-policy replay

Preserve:

- apparent competence
- output pattern
- first-order success when possible

Destroy:

- live adaptation
- state-contingent correction
- endogenous update coupling

Expected behavior:

- first-order performance may remain stable, while second-order correction structure should degrade or separate.

### 4. State-independent forcing

Preserve:

- externally imposed control profile
- possible output smoothness

Destroy:

- conditional dependence on current state or belief

Expected behavior:

- if curvature structure depends on state-conditional control, state-independent forcing should separate from the observed condition.

### 5. Observer expansion without authority

Preserve:

- additional observations or richer logging

Destroy or withhold:

- actual control authority
- intervention capacity
- endogenous regulation

Expected behavior:

- more observation alone should not create owned correction.

### 6. Noise and masking tests

Introduce:

- observation noise
- partial observability
- masking
- degradation

Expected behavior:

- robust diagnostics should degrade transparently rather than invent strong signal.

### 7. Matched-mean control

Preserve:

- mean performance
- comparable first-order summaries

Test:

- whether second-order structure still separates regimes

Expected behavior:

- if the diagnostic is useful, it should add information beyond matched first-order performance.

## Required Negative-Control Rule

Every promoted ∆∆F diagnostic claim should say what negative control was used and what result would have invalidated the claim.

## Unsafe Pattern

Avoid:

> Curvature appeared, therefore the system is adaptive.

Use:

> Curvature structure separated from the specified null under the tested perturbation conditions, subject to stated limitations.
