# Pilot Readiness Policy

Status: scaffold  
Date: 2026-06-28

## Purpose

Score whether a buyer-safe pilot should proceed, be rescoped, require redaction, require legal/CPA support, or be declined.

## Possible outputs

- READY
- NOT_READY
- NEEDS_SCOPING
- NEEDS_REDACTION
- NEEDS_COUNSEL
- DECLINE

## READY requires

- workflow is bounded
- buyer can export observable traces
- buyer retains labels
- human adjudication is available
- success criteria exist
- prohibited materials are excluded
- timeline is realistic
- data boundaries are safe
- no legal/compliance/certification claim
- no free pilot expectation

## NEEDS_COUNSEL triggers

- procurement
- contract terms
- paid pilot
- licensing
- exclusivity
- acquisition
- IP assignment
- data transfer
- money movement

## DECLINE triggers

- buyer asks for compliance certification
- buyer asks for automated verdict
- buyer asks for production validation
- buyer asks for truth validation
- buyer asks for legal/financial conclusions

## Boundary

This policy is an internal buyer-safe readiness scaffold only.

It does not create legal advice, financial advice, binding terms, compliance certification, production validation, automated verdicts, free-pilot commitments, autonomous client action, truth validation, or hidden-mechanism claims.
